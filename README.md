# frostrelay

Минимальная замена Icecast для одной станции: принимает поток от BUTT
(или любого клиента с Icecast source-протоколом), раздаёт его слушателям
и параллельно пишет резервную копию эфира на диск. Один бинарник,
только стандартная библиотека Go.

## Сборка

```sh
go build -ldflags="-s -w" -o frostrelay .
```

Кросс-компиляция для сервера на Linux:

```sh
GOOS=linux GOARCH=amd64 go build -ldflags="-s -w" -o frostrelay-linux .
```

## Запуск

```sh
./frostrelay -password СЕКРЕТ
```

Флаги (все необязательные, кроме пароля):

| Флаг        | По умолчанию | Что делает                                  |
|-------------|--------------|---------------------------------------------|
| `-listen`   | `:8000`      | адрес и порт HTTP-сервера                   |
| `-mount`    | `/stream`    | точка монтирования                          |
| `-password` | —            | пароль источника (или `FROSTRELAY_PASSWORD`)|
| `-archive`  | `archive`    | каталог записей эфиров                      |
| `-name`     | `frostrelay` | название станции на странице плеера         |

## Прод в Docker (за реверс-прокси Caddy)

Схема для сервера с несколькими проектами: общая docker-сеть `web`,
в ней Caddy принимает 80/443 и проксирует в контейнеры по именам.

Один раз на сервере:

```sh
docker network create web
```

Запуск frostrelay:

```sh
cp .env.example .env   # вписать пароль источника
docker compose up -d --build
```

Что получается:

- образ собирается из исходников (multi-stage, итог ~6 МБ из scratch);
- записи эфиров падают в `./archive` рядом с compose-файлом;
- порт `8000` опубликован наружу — это прямой вход для BUTT
  (источник идёт мимо Caddy, без TLS);
- слушатели ходят через Caddy — фрагмент конфига в `Caddyfile.example`
  (важна настройка `flush_interval -1`, чтобы Caddy не буферизовал поток);
- `TZ` в `.env` задаёт часовой пояс для имён файлов архива.

В BUTT при этом: адрес — IP/домен сервера, порт `8000`, mountpoint
`stream`, user `source`.

## Настройка BUTT

Settings → Main → Server: тип **Icecast**, адрес и порт сервера,
password — ваш пароль, mountpoint — `stream` (без слэша), user — `source`.

## Как это устроено

- `/` — единственная страница с плеером; раз в 5 секунд опрашивает
  `/status.json`, при появлении эфира кнопка оживает.
- `/stream` — `PUT`/`SOURCE` от источника (один одновременно),
  `GET` для слушателей. Новому слушателю сразу отдаётся буфер последних
  128 КБ, чтобы звук стартовал без паузы.
- `/status.json` — `{live, listeners, song, name, mount}`.
- `/admin/metadata` — приём «сейчас играет» от BUTT.
- Каждый эфир пишется в `archive/ГГГГ-ММ-ДД_ЧЧ-ММ-СС.mp3`
  (расширение — по Content-Type потока).

Слушатель, который не успевает читать поток (буфер 64 чанка по 8 КБ),
отключается, чтобы не задерживать остальных. Если источник молчит
60 секунд, соединение считается мёртвым и эфир завершается.
